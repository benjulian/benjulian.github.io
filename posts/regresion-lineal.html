Linear Regression Under the Hood

9 of October, 2025

Introduction

One of the most fundamental machine learning models is Linear Regression.

Even though it is a super simple model, it allows us to create a mathematical model that can predict outcomes based on its inputs. Although it has a lot of relation with statistics and probability, in this post we will focus on the pure math mechanics of Linear Regression.

In order to achieve a complete understanding of the content of this post, the reader must be familiar with the following branches of mathematics:

  • Linear Algebra (as this is the language of machine learning models);
  • Calculus (as it is the language of change);
  • Optimization (as it is how the model "learns")

Model Architecture

Let us start by picturing how to architecture of the model is. From this, we can take a look at the following image:

[HERE WILL BE AN IMAGE]

As the image shows, the architecture is rather simple.

First, we multiply each of the "feature" values (mathematically, those are variables, but in the context of machine leaning, we call them features) with its corresponding "weight" value (again, technically those are variables, but in this context, we call them weights), sum those products up and then add a last value, called "bias".

That final value is the corresponding prediction of our model based on the inputs.

Data

IMPORTANT NOTE: in these posts, ALL of the vectors will be treated as matrices.

First, even though we will analyze the model as we only have one sample with one single feature (something that is almost never the case in machine learning), the fact that we will treat each vector as matrices will, intuitively, let you understand how it works if there are more samples and / or features.

Feature data

The main data we collect is the so called "feature data", that is represented with an X.

X will be represented as a matrix, with the first index "a" meaning "number of samples" and the second index "b" meaning "number of features".

It is like this:

\[ X = \begin{bmatrix} 3 \end{bmatrix} \rightarrow X \in \mathbb{R}^{1(a) \times 1(b)} \]

Objective value data

The corresponding data associated with the feature data contains the value at which the model should aim. This data is represented with a Y.

Y will be represented as a matrix also, with the first index "a" meaning "number of samples" and the second index "c" meaning the "number of objective values".

It is like this:

\[ Y = \begin{bmatrix} 3 \end{bmatrix} \rightarrow Y \in \mathbb{R}^{1(a) \times 1(c)} \]
